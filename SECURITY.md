# Security Policy / Política de Segurança

## Supported Versions / Versões Suportadas

| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Yes    |
| < 1.0   | ❌ No     |

## Reporting a Vulnerability / Reportando Vulnerabilidades

**English:**

If you discover a security vulnerability in any Infomance repository, please report it responsibly:

1. **Do NOT** create a public GitHub issue
2. Email us at **security@infomance.com.br**
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will acknowledge receipt within 48 hours and provide a detailed response within 7 days.

**Português:**

Se você descobrir uma vulnerabilidade de segurança em qualquer repositório da Infomance, por favor reporte de forma responsável:

1. **NÃO** crie uma issue pública no GitHub
2. Envie email para **security@infomance.com.br**
3. Inclua:
   - Descrição da vulnerabilidade
   - Passos para reproduzir
   - Impacto potencial
   - Sugestão de correção (se houver)

Confirmaremos o recebimento em até 48 horas e forneceremos uma resposta detalhada em até 7 dias.

## Security Best Practices / Boas Práticas de Segurança

When using our SDKs:

- **Never** commit API keys to version control
- Use environment variables for credentials
- Keep SDKs updated to the latest version
- Enable SSL verification in production (default)

Ao usar nossos SDKs:

- **Nunca** faça commit de API keys no controle de versão
- Use variáveis de ambiente para credenciais
- Mantenha os SDKs atualizados para a última versão
- Mantenha a verificação SSL habilitada em produção (padrão)
